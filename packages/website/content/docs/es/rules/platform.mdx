## vercel-edge-heavy-import

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-edge-heavy-import`</Callout>

Detecta archivos de runtime edge que importan dependencias pesadas o centradas en Node como `node:fs`, `node:crypto`, `sharp`, o `@aws-sdk/*`.

**Por quﾃｩ importa:** Las funciones edge tienen lﾃｭmites estrictos de tamaﾃｱo y ejecuciﾃｳn. Las dependencias pesadas de Node.js aumentan el tiempo de cold start y pueden fallar en runtime.

**Correcciﾃｳn:** Mueve la lﾃｳgica pesada a funciones de runtime Node.js o trabajos en segundo plano, y mantﾃｩn los manejadores edge ligeros.

## vercel-sequential-database-await

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-sequential-database-await`</Callout>

Detecta rutas API con 3 o mﾃ｡s llamadas secuenciales a Prisma o base de datos sin `Promise.all`.

**Por quﾃｩ importa:** Cada llamada secuencial a la base de datos aﾃｱade latencia a la ejecuciﾃｳn de la funciﾃｳn. Paralelizar consultas independientes reduce la duraciﾃｳn total y el costo.

```ts title="Bad"

const posts = await prisma.post.findMany();
const tags = await prisma.tag.findMany();
```

```ts title="Good"
const [users, posts, tags] = await Promise.all([
  prisma.user.findMany(),
  prisma.post.findMany(),
  prisma.tag.findMany(),
]);
```

## vercel-large-static-asset

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-large-static-asset`</Callout>

Detecta recursos estﾃ｡ticos (imﾃ｡genes, fuentes, vﾃｭdeos, PDFs) de 4 KB o mﾃ｡s servidos desde tu repositorio de la app.

**Por quﾃｩ importa:** Los archivos estﾃ｡ticos grandes servidos desde tu despliegue Vercel consumen ancho de banda en cada peticiﾃｳn. Moverlos a una CDN o almacenamiento de objetos dedicado (Cloudflare R2, S3) reduce los costos de ancho de banda.

Reporta hasta 20 archivos, ordenados por tamaﾃｱo (los mﾃ｡s grandes primero).

## vercel-consider-bun-runtime

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-consider-bun-runtime`</Callout>

Detecta proyectos no configurados para el runtime Bun (sin `packageManager: "bun@..."` en `package.json` y sin archivo `bun.lock`).

**Por quﾃｩ importa:** El runtime Bun puede reducir la sobrecarga de instalaciﾃｳn y compilaciﾃｳn en Vercel en comparaciﾃｳn con Node.js.

**Correcciﾃｳn:** Revisa la [guﾃｭa del runtime Bun](https://vercel.com/docs/functions/runtimes/bun) y cambia si tu proyecto es compatible.

## vercel-avoid-platform-cron

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-avoid-platform-cron`</Callout>

Detecta `crons` configurados en `vercel.json`.

**Por quﾃｩ importa:** Los cron jobs de Vercel se ejecutan como funciones serverless, facturadas por ejecuciﾃｳn. Las cargas de trabajo programadas con patrones predecibles suelen ser mﾃ｡s econﾃｳmicas usando GitHub Actions o Cloudflare Workers Cron Triggers.

## vercel-consider-fluid-compute

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-consider-fluid-compute`</Callout>

Detecta proyectos con 3 o mﾃ｡s rutas API/servidor.

**Por quﾃｩ importa:** Fluid Compute mejora la concurrencia y reduce la sobrecarga de ejecuciﾃｳn para cargas de trabajo con latencia variable o trﾃ｡fico intermitente. Vale la pena evaluarlo para proyectos con mﾃｺltiples rutas de servidor.

## vercel-suggest-turbopack-build-cache

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-suggest-turbopack-build-cache`</Callout>

Esta comprobaciﾃｳn depende de la versiﾃｳn y solo se aplica a proyectos con Next.js 16+.

Detecta archivos `next.config` con configuraciﾃｳn `experimental` pero sin `turbopackFileSystemCacheForBuild`.

**Por quﾃｩ importa:** Next.js 16+ soporta la cachﾃｩ de compilaciﾃｳn de Turbopack, que puede reducir significativamente los tiempos de compilaciﾃｳn.

```js title="Good"

  experimental: {
    turbopackFileSystemCacheForBuild: true,
  },
};
```

## vercel-suggest-deploy-archive

<Callout type="warn">Warning ﾂｷ `vercel-doctor/vercel-suggest-deploy-archive`</Callout>

Detecta proyectos con 5.000 o mﾃ｡s archivos.

**Por quﾃｩ importa:** Los proyectos grandes pueden alcanzar lﾃｭmites de API durante el despliegue. Usar el modo archive sube un ﾃｺnico tarball en lugar de archivos individuales, reduciendo el tiempo de despliegue aproximadamente un 50%.

```bash

```
